# Access Quest: Caça aos Bugs de Acessibilidade 

## 1. Descrição do Recurso
* **Breve descrição:** Um jogo interativo de auditoria web estilo "Point and Click" onde o jogador assume o papel de um analista de acessibilidade digital. O objetivo é encontrar 8 erros comuns de acessibilidade baseados nas diretrizes da WCAG em uma interface que simula um e-commerce real.
* **Público-Alvo:** Estudantes de tecnologia, desenvolvedores web iniciantes, designers de interface (UX/UI) e entusiastas de inclusão digital.
* **Objetivos Educacionais:** Ensinar de forma prática e lúdica a identificar barreiras de acessibilidade na web, compreender o impacto dessas falhas na vida de usuários reais e aprender a corrigi-las utilizando código semântico.
* **Características de Acessibilidade do Próprio Jogo:** * Totalmente navegável via teclado (teclas Tab, Enter e Space);
  * Foco visual altamente visível para navegação sem mouse;
  * Respeita o uso de leitores de tela com tags semânticas e atributos ARIA;
  * Modais de feedback acessíveis com controle de foco.
